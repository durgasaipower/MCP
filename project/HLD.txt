================================================================================
SECTION 1: OVERVIEW
================================================================================

A self-hosted MCP server that allows clients (LLMs, agents, IDEs) to:
    1. Search documentation via RAG (semantic vector search)
    2. Browse/read documentation via a virtual filesystem
    3. Upload new documents via a REST endpoint (non-MCP)

The system ingests documents into S3, automatically chunks and embeds them
into a vector database, and exposes the content through MCP tools.

SECTION 4: COMPONENT DETAILS
================================================================================

───────────────────────────────────────────────────────────────────────────────
4.1 MCP SERVER
───────────────────────────────────────────────────────────────────────────────

    Runtime:        Python (FastMCP + uvicorn)
    Deployment:     ECS Fargate (or Lambda with streaming)
    Port:           8000
    Endpoints:
        POST /mcp       → MCP protocol handler
        POST /upload    → File upload REST endpoint
        GET  /health    → Health check

    Tools exposed:
        1. rag_search(query: str) → str
        2. query_filesystem(command: str) → str


───────────────────────────────────────────────────────────────────────────────
4.2 TOOL 1: rag_search
───────────────────────────────────────────────────────────────────────────────

    Purpose:    Semantic search across all uploaded documentation
    Input:      {"query": "string"}
    Output:     Markdown-formatted results with titles, snippets, paths

    Internal flow:
        1. Receive query string
        2. Generate embedding using same model as ingestion
        3. Query vector DB for top-K similar chunks (k=5 default)
        4. Format results: title, snippet, source path, relevance score
        5. Return as markdown text

    Vector DB schema per chunk:
        {
            "id": "uuid",
            "embedding": [float, ...],    ← 1536-dim (OpenAI) or similar
            "text": "chunk content...",
            "metadata": {
                "source_path": "/guides/quickstart.md",
                "title": "Quickstart Guide",
                "heading": "## Installation",
                "chunk_index": 3,
                "uploaded_at": "2026-05-24T12:00:00Z"
            }
        }


───────────────────────────────────────────────────────────────────────────────
4.3 TOOL 2: query_filesystem
───────────────────────────────────────────────────────────────────────────────

    Purpose:    Read/browse documentation files directly
    Input:      {"command": "string"}
    Output:     Plain text command output

    Supported commands:
        tree [path] -L [depth]      → directory structure
        ls [path]                   → list directory
        cat [path]                  → read full file
        head -N [path]              → first N lines
        tail -N [path]              → last N lines
        find [path] -name [pattern] → find files
        rg [pattern] [path]         → regex search across files
        wc [path]                   → word/line count

    Data source:
        - Reads from S3 (with local cache for performance)
        - Uses index.json to know file structure
        - Stateless: each call starts fresh at /

    index.json format (stored in S3):
        {
            "files": [
                {"path": "/guides/quickstart.md", "title": "Quickstart", "size": 4523},
                {"path": "/guides/advanced.md", "title": "Advanced Guide", "size": 8901},
                {"path": "/api-reference/endpoints.md", "title": "API Endpoints", "size": 3200}
            ],
            "last_updated": "2026-05-24T12:00:00Z",
            "total_files": 3
        }


───────────────────────────────────────────────────────────────────────────────
4.4 UPLOAD ENDPOINT (REST, not MCP)
───────────────────────────────────────────────────────────────────────────────

    Endpoint:   POST /upload
    Auth:       API key in header (X-API-Key)
    Content:    multipart/form-data

    Request:
        Headers:
            X-API-Key: <api-key>
            Content-Type: multipart/form-data

        Body:
            file: <binary file content>
            path: "/guides/new-guide.md"       (target path in filesystem)
            title: "New Guide"                 (optional metadata)

    Response:
        {
            "status": "uploaded",
            "path": "/guides/new-guide.md",
            "s3_key": "documents/guides/new-guide.md",
            "message": "File uploaded. Indexing will complete in ~30 seconds."
        }

    Internal flow:
        1. Validate API key
        2. Upload file to S3 at documents/<path>
        3. Update index.json in S3
        4. Return success (indexing happens async via S3 trigger)


───────────────────────────────────────────────────────────────────────────────
4.5 INGESTION PIPELINE (Lambda / Fargate)
───────────────────────────────────────────────────────────────────────────────

    Trigger:    S3 PutObject event on docs-mcp-content bucket
    Runtime:    Python Lambda (or Fargate for large files)

    Steps:
        1. Receive S3 event → get bucket + key
        2. Download file from S3
        3. Detect format (md, mdx, txt, pdf)
        4. Parse content → extract text
        5. Chunk text:
            - Split by headings (## / ###)
            - Max chunk size: 1000 tokens
            - Overlap: 100 tokens
        6. Generate embeddings for each chunk
            - Model: OpenAI text-embedding-3-small (or Bedrock Titan)
        7. Upsert into vector DB:
            - Delete old chunks for same source_path (replace)
            - Insert new chunks with metadata
        8. Log completion


───────────────────────────────────────────────────────────────────────────────
4.6 VECTOR DATABASE
───────────────────────────────────────────────────────────────────────────────

    Options (pick one):
        - Amazon OpenSearch Serverless (with vector engine)
        - PostgreSQL + pgvector (on RDS)
        - Pinecone (managed, external)

    Schema:
        Collection/Index: "doc_chunks"
        Fields:
            id:          UUID
            embedding:   vector(1536)
            text:        string (chunk content)
            source_path: string ("/guides/quickstart.md")
            title:       string
            heading:     string (section heading)
            chunk_index: integer
            uploaded_at: timestamp
